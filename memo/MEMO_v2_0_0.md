# 自販機プログラム仕様書 (v2.0.0)

## 1. 機能 (Features)
### サイドバー (Sidebar)
- **金銭投入管理**
    - 10円〜10000円の各貨幣を選択して投入。
    - 貨幣ごとの投入枚数に応じた上限設定（安全装置）。
- **液晶画面（メッセージパネル）**
    - 状態（待機・購入成功・残高不足・返却）に応じたメッセージ表示。
- **コイン返却口（物理返却シミュレーション）**
    - **[New!] 各貨幣の内訳による返却**: 単なる合計額ではなく「どの硬貨が何枚出るか」を算出して表示。

### メイン画面 (Main Screen)
- **商品ラインナップ表示**
    - 商品名・価格の多段グリッド表示。
- **購入処理**
    - 商品選択後の購入ボタン押下によるイベント実行。

### 管理ロジック (Back-end Logic)
- **[New!] 金庫管理 (CashManagerクラス)**
    - 以下の2つの状態を管理する：
        - `inventory`: 自販機内の貨幣在庫
    - **お釣り計算アルゴリズム (貪欲法)**:
        - 高額貨幣から優先的に払い出し
        - 機内在庫（inventory）と照らし合わせて算出
        - 払い出し不可な場合はエラーとする

- **[New!] 履歴管理 (Transaction Log)**
    - すべてのアクション（投入・購入・返却）を構造化データ(dict)として記録。
    - **記録項目**:
        - `datetime`
        - `type` (deposit/purchase/refund)
        - `amount`
        - `item`
        - `change_detail`
        - `balance_after`（処理後残高）

- **現金以外の決済方法**（ICカード、クレジットカードなど）New! -> 保留（実装予定なし）
- **お金投入前の選択式** New! -> 保留（実装予定なし）

## 2. 非機能 (Non-Functional)
- **バリデーション**
    - お金投入前の購入阻止、商品未選択時の警告、金額不足時のエラー表示。
- **[New!] 払い出し制約**
    - お釣りが機内在庫不足で払い出せない場合：
        - 購入処理は失敗とする（返却可能）
- **拡張性 (Scalability)**
    - 履歴データを `st.session_state` にリスト保持し、将来的に Pandas DataFrame へ変換可能にする。
    - CSV保存やDB連携を見越し、ロジック層(CashManager)にUI操作(Streamlit)を混入させない。

---

## 3. 詳細な流れ (Detailed Sequence)
1. **初期化 (Initialization)**
    - 自販機内の各貨幣（10円〜10000円）の初期在庫枚数を設定。
2. **待機状態 (Idle)**
    - メッセージパネルに「いらっしゃいませ」を表示。
3. **金銭投入 (Deposit)**
    - 投入された貨幣は `inventory` に加算する。
    - 投入イベントを履歴に記録。->保留
4. **購入実行 (Purchase)**
    - **成功時**: 
        - `inserted_cash` + 商品価格を基にお釣りを計算
        - お釣りを算出後、`inventory` に投入分を反映
        - 返却分を `inventory` から減算
        - 購入イベントを履歴に記録
    - **失敗時**:
        - 残高不足 または 在庫不足の場合は処理中断
5. **返却実行 (Refund)**
    - `inserted_cash` をそのまま貨幣単位で返却する
    - `inventory` には影響を与えない
    - 履歴を記録

---

## 4. 課題・設計上の約束事 (Review Standards)
- **戻り値の構造化**: お釣り計算結果等は、`{100: 1, 10: 3}` のような辞書型(dict)で受け渡す。
- **責務の分離**: `CashManager` は計算と状態管理に徹し、メッセージ表示や描画は行わない。
- **データ分析への備え**: 履歴ログのキー名は、将来Pandasで扱う際の列名として適切な名称(英名)を使用する。

## 5. フォルダ構成
```
vending_machine_app/
├── src/
│   ├── app.py
│   ├── cash_manager.py
│   └── data_handler.py
└── ...
```
**各ファイルの説明**
- app.py: **UI（Streamlit）**を担当します。画面のレイアウトやユーザー操作の受け付けのみを行い、複雑な計算ロジックは書きません。

- cash_manager.py: **金庫管理（CashManagerクラス）**を担当します。お釣り計算（貪欲法）や投入金の管理など、このアプリの「心臓部」となるロジックをここに集約します。

- data_handler.py: **データ入出力（CSV/Pandas）**を担当します。履歴の保存や読み込みなど、ファイル操作に関する処理を専門に行います。
